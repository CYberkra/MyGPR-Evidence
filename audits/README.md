# Evidence Audit Notes

Audit notes document whether a committed evidence package is acceptable for
research discussion, group meetings, or paper drafting.

Each audit note should include:

- Checked source commit.
- Checked evidence commit.
- Result: `pass`, `basic pass`, or `fail`.
- P0 / P1 / P2 issues.
- Whether B-scan figures were inspected.
- Whether baseline sanity warnings were checked.
- Whether heuristic QC metrics and ground-truth metrics were clearly separated.
- Whether stress/demo profiles were labeled as stress/demo, not field-flight
  baselines.

Use one concise Markdown file per audit when the audit has durable value.
